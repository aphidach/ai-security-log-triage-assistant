Error Taxonomy

Normal boundary failed

normal ได้ 0/5 อีกครั้ง เคส single failed login และ health check ยังถูกยกระดับเป็น suspicious/high

Attack-type boundary failed

SQLi และ traversal อย่างละ 4/5 ถูกลากไป failed_login_bruteforce ทั้งที่ evidence มี payload/path signal ชัด

Contract improved, not enough

JSON/schema ดีขึ้นจาก 0.92 เป็น 0.96 แต่ปัญหาหลักคือ semantic mapping ไม่ใช่ output shape แล้ว

Recommended Next Actions

  1. ยืนยันว่า vLLM endpoint โหลด adapter v3.1 จริง โดยใช้ served model name แยก เช่น lfm2-security-triage-v3-1 และบันทึก adapter path ใน report metadata
  2. ทำ hard-contrast memorization probe กับ data/generated/v3-hard-contrast-security-triage.jsonl เพื่อดูว่า model จำ training supplement ได้หรือไม่
  3. ถ้าจำ hard contrast ยังไม่ได้ ให้ทำ v3.2 training recipe ที่แรงขึ้น เช่นเพิ่ม steps/epochs หรือเพิ่ม LoRA rank ก่อนสรุปว่า data coverage ไม่พอ
  4. ถ้าจำ hard contrast ได้แต่ mini ยังพัง ให้เพิ่ม coverage ตาม mini failure โดยเฉพาะ normal hard negatives, SQLi login payload, encoded traversal และ nmap/SYN scan cases
  5. ยังไม่ใช้ fixed data/splits/test.jsonl จนกว่า mini semantic eval จะผ่าน gate ขั้นต่ำ
